Quick Start

Get started with Liana using the default configuration setup. No custom providers, no extra configuration.

1. Add Liana to your project

Liana is not yet published to Maven Central. Installation instructions will be available soon.

2. Create a configuration file

By default, Liana looks for a file named application in the classpath, using any supported format.

Example: application.yaml

app:
  name: Liana
server:
  port: 8080

3. Load the configuration

Use the default ConfigurationManager and ResourceLocation to load configuration data. 

ResourceLocation location = ResourceLocation.builder().build();

ConfigurationManager manager = ConfigurationManager.builder().build();

Configuration config = manager.load(location);

4. Read values

Access configuration values using type-safe getters. You can always provide a default value. 

String appName = config.getString("app.name", "DefaultApp");

int port = config.getInt("server.port", 8080);

How defaults work

Without any customization, Liana applies the following defaults:

  • Provider: classpath
  • Base resource name: application
  • Profile variable: profile
  • Default profile: default
  • Profile environment variable: LIANA_PROFILE

This means Liana will automatically try to load: 

application.(properties|yaml|json|xml)
application-${profile}.(properties|yaml|json|xml)

If LIANA_PROFILE is not set, the profile default is used.

Continue with Architecture to understand how configuration is resolved, or jump to Advanced Usage for custom providers and loaders.
