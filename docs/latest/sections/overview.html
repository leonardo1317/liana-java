Liana Configuration

Liana is a lightweight, framework-agnostic Java configuration library designed for clarity, predictability, and extensibility.

Modern Java applications often need to load configuration from multiple sources, formats, and environments. Existing solutions tend to be either tightly coupled to a framework or overly complex for simple use cases.

Liana provides a deterministic configuration resolution pipeline with sensible defaults, while allowing you to extend every step when needed. It adapts to your application — not the other way around.

Why Liana?

  • Framework-agnostic: works in plain Java, microservices, CLI tools, and any JVM-based environment.
  • Multi-format support: load and merge configuration from properties, YAML, JSON, and XML out of the box.
  • Predictable overrides: later resources override earlier ones, making environment layering explicit and safe.
  • Deep placeholder interpolation: resolve variables across nested structures with support for defaults.
  • Type-safe access: read values as primitives, collections, maps, or strongly-typed POJOs.
  • Immutable and thread-safe: configuration is read-only after loading.

Designed for Extension

Liana is built around a small set of well-defined abstractions: providers, loaders, and a resolution pipeline. This makes it easy to introduce custom configuration sources (filesystem, HTTP, secrets managers) or new file formats without changing the core.

You can mix multiple providers in the same configuration load and even assign a specific provider per resource when needed.

What Liana Is — and Is Not

  • ✔ Liana is a focused configuration library with explicit behavior and minimal magic.
  • ✘ Liana is not a full application framework or a dependency injection container.
  • ✔ Liana complements frameworks like Spring, Micronaut, or Quarkus when you want more control over configuration resolution.
