API Reference

This section documents detailed behaviors, optional methods, and exceptions not fully covered in usage examples. It serves as a technical reference for developers who need exact API capabilities.

ConfigurationManager

  • load(ResourceLocation location) – loads configuration from the specified location. Throws: NullPointerException if location is null. Notes: Returns immutable Configuration. Duplicate resources are merged in resolution order.
  • builder() – creates a builder for constructing a ConfigurationManager instance. Notes: Applies defaults for providers and loaders.

ConfigurationManagerBuilder

  • addProviders(ResourceProvider... providers) – registers one or more providers. Notes: Providers must have unique keys; duplicates are ignored or overridden.
  • addLoaders(ResourceLoader... loaders) – registers one or more loaders. Notes: Loaders are selected based on file extensions; parsing exceptions propagate as RuntimeException.
  • build() – returns an immutable ConfigurationManager. Throws: IllegalStateException if mandatory components are missing.

ResourceLocation

  • provider() – returns provider identifier; default: "classpath".
  • baseDirectories() – ordered set of directories used for relative path resolution.
  • resourceNames() – logical resource names; duplicates resolved in order.
  • variables() – placeholder variables; Notes: invalid keys may throw InvalidVariablesException.
  • verboseLogging() – indicates whether detailed logging is enabled.
  • placeholder() – returns the placeholder strategy; default uses ${...} syntax.
  • builder() – creates a ResourceLocationBuilder for constructing instances.

ResourceLocationBuilder

  • provider(String provider) – sets the provider; default is "classpath".
  • baseDirectories(String... baseDirectories) – sets ordered directories; order affects resolution.
  • addResource(String resourceName) – adds a single resource to resolve.
  • addResources(String... resources) – adds multiple resources to resolve.
  • addResourceFromList(List<String> resources) – adds resources from a list.
  • addVariable(String key, String value) – adds a single variable; invalid pairs throw InvalidVariablesException.
  • addVariables(String... variables) – adds alternating key/value pairs.
  • addVariablesFromMap(Map<String,String> variables) – adds variables from a map.
  • verboseLogging(boolean verboseLogging) – enables/disables detailed logging.
  • placeholders(Placeholder placeholder) – sets the placeholder engine; defaults applied if missing.
  • build() – returns an immutable ResourceLocation instance.

Configuration

  • containsKey(String key) – checks existence of a key.
  • get / getOrThrow – type-safe access. Throws: NoSuchElementException for required values. Notes: Generic overloads support lists and maps; fallback defaults can be provided.
  • getList / getMap – converts sections into collections. Notes: Empty or missing sections return empty collection if default provided.
  • getRootAsMap / getRootAs(Type) – returns the full configuration tree. Notes: Returns unmodifiable map; type conversion may throw IllegalArgumentException if incompatible.
  • getString / getInt / getBoolean / getDouble / getDuration – retrieves typed values, optionally with default.

ResourceProvider / ResourceLoader

  • resolveResource / load – core methods to obtain configuration content; runtime exceptions may occur on missing file, parse failure, or invalid content.
  • validateResource(...) – optional validation; implementers may override to enforce stricter rules.
  • keys() – returns unique provider identifiers or supported file extensions.

Placeholder / PlaceholderBuilder

  • replaceIfAllResolvable – resolves placeholders; returns Optional.empty() if any cannot be resolved. Notes: Supports extra sources or maps for dynamic resolution.
  • builder() – allows configuring prefix, suffix, delimiter, and escape character. Notes: Default syntax: ${key:default}.
